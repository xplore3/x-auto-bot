Agent Knowledge Base

Agent 知识库与行为规则

这里不是让新用户手写 Prompt，而是沉淀 Agent 的长期记忆：它是谁、服务谁、讲什么、怎么讲、哪些话不能说。

Auto Saved
01 定位与入口

决定读者为什么关注，以及关注后应该去哪里。

02 读者地图

决定选题、案例、术语密度和互动对象。

03 内容引擎

决定每日输出的观点、故事、素材和选题结构。

04 表达边界

决定语气、风险红线和评论引流方式。

定位与行动入口 用户看见账号后，应该立刻理解你是谁、凭什么值得关注、下一步做什么。
读者地图与痛点 把“目标用户”从泛泛人群拆成可写、可互动、可转化的具体读者。
内容引擎 把日常输入变成稳定输出，避免 Agent 每次临场发挥。
表达方式、边界与互动 让 Agent 知道怎么像你说话，也知道哪些内容不能碰。
专家模式：回复 Prompt 仅在你明确要调整自动回复底层指令时修改。

Test & Review Center

测试发帖、复盘与策略迭代

这里负责把 Agent 从“能生成内容”推进到“能稳定改进”：先验证发布链路，再记录表现，最后把有效信号喂回知识库。

Live Test
1 预演

检查文案换行、中文预填和账号登录态。

2 小流量发布

先用测试推文验证真实发送链路。

3 记录信号

记录涨粉、回复、收藏、误解和高表现角度。

4 迭代策略

把有效模式写回选题、语气和互动规则。

Dry Run

测试发帖链路

不依赖 API Key。后台会在发出前统一做中文换行、文本校验和失败暂停。

Safety Checks

发送前检查项

  • 是否存在 X 登录态
  • 输入框文本是否与待发文本一致
  • 中文和换行是否保留
  • 发送按钮是否由 X 自身判定可用
  • 失败时是否暂停并写入日志
